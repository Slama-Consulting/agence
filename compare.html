L'atelier · Méthode & principes

On ne construit pas un agent
comme on bricole une démo.

Chaque agent de la galerie suit la même discipline : un seul contexte de conversation, des secrets qui ne quittent jamais le poste, un protocole MCP propre, et une distribution servie depuis GitHub Pages.

01

Single-context strict

L'agent ne harcèle pas l'utilisateur question après question. Toute la clarification se fait en un seul tour ; ce qui manque devient une hypothèse documentée dans le rapport.

02

Secrets sur le poste

Tokens Jira, GitHub, Artifactory, LLM — tout vit dans ~/.bughound/.env. Aucun proxy, aucun service. chmod 600.

03

MCP-first

Les agents s'enregistrent comme serveurs MCP dans Claude Code, Copilot et Cursor. Pas d'API maison, pas de chat captif — vous gardez votre IDE, votre LLM, votre flux.

04

Mode best par défaut

BugHound court-circuite ses propres appels LLM et délègue au LLM externe avec un prompt prêt à exécuter. Votre fenêtre de contexte respire ; les modes legacy / monolithique restent disponibles.

05

Drafts groupés

Pour les agents qui interagissent avec une forge (WaliPR), les findings arrivent en drafts « Add to review ». Une seule notification au reviewé, jamais N spam séparés.

06

Distribution GitHub Pages

Le code source des agents reste privé. Seule la wheel et install.sh sont servis publiquement — curl … | bash, et c'est en place.

Cycle d'une enquête

  1. Ingestion déterministe

    BugHound récupère le ticket Jira, OCR les screenshots, slice les logs, repère les artefacts. Aucun LLM impliqué ici — c'est du Python pur, reproductible.

  2. Élicitation structurée (si besoin)

    S'il manque des infos, l'agent regroupe toutes ses questions en un seul message via AskUserQuestion (Claude Code) ou un markdown formaté ailleurs. Si tout est clair, on enchaîne sans interrompre.

  3. Diagnostic dans votre LLM

    Le prompt prêt-à-exécuter passe la main à votre LLM externe (Claude, GPT, Copilot…). C'est lui qui produit le raisonnement — BugHound a juste préparé le terrain proprement.

  4. Rapport actionnable

    Cause racine (FR + EN ≤ 80 mots pour le PO), reproduction étape par étape, fix proposé avec niveau de confiance, zones à risque de régression. Pas de jargon inutile, pas de paragraphes de remplissage.
