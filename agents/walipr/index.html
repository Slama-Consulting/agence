WaliPR

Inspecteur de pull requests

« Je relis chaque diff comme un dossier sensible. Mes remarques se posent là où elles font mouche : sur la ligne. »

Compatible : Claude Code Copilot Cursor

Mission

WaliPR relit vos pull requests et merge requests sur GitLab, GitHub et Bitbucket (Cloud + Server). Il analyse le diff Kotlin / Python / TypeScript / Java, traque les régressions, les fuites de ressources et les anti-patterns, puis pose ses remarques en commentaires inline, ligne par ligne, avec des patches prêts à appliquer quand c'est pertinent.

Capacités

  • review_local_diff Review d'un diff git local (working tree, branche, range).
  • review_staged Review des changements stagés avant git commit.
  • review_merge_request Review d'une PR/MR distante par URL — GitLab, GitHub, Bitbucket. Pose les commentaires inline directement sur la forge.

Installer WaliPR

WaliPR est livré avec BugHound. Une seule commande installe l'agent et l'enregistre comme serveur MCP walipr dans votre IDE. 

curl -fsSL https://slama-consulting.github.io/agence/install.sh | bash
bughound mcp-install --agents walipr

Pour installer WaliPR et SherleKhomes en même temps : bughound mcp-install --agents sherlekhomes,walipr.

Configurer votre forge

Pour que WaliPR puisse lire et commenter une PR/MR, il a besoin d'un token de la forge concernée. Les tokens vivent dans ~/.bughound/.env, sur votre machine — jamais dans un dépôt git. 

# GitLab (self-managed ou gitlab.com) — scope: api
# WALIPR_GITLAB_TOKEN=

# GitHub (github.com ou GHE) — scopes: repo, write:discussion
# WALIPR_GITHUB_TOKEN=

# Bitbucket Cloud — App Password : pull requests (read+write)
# WALIPR_BITBUCKET_USER=mon-username
# WALIPR_BITBUCKET_TOKEN=

# Bitbucket Server / Data Center — HTTP Access Token : PR (read+write)
# WALIPR_BITBUCKET_SERVER_TOKEN=

Vous n'avez besoin que du token correspondant à la forge que vous utilisez. Les autres lignes restent commentées.

Premiers pas

  1. Review en lecture seule d'une PR distante : bughound review-mr https://github.com/org/repo/pull/123
  2. Review d'une MR GitLab avec commentaires inline postés sur la forge : bughound review-mr <url> --post-inline
  3. Review locale avant push : bughound review-staged
  4. Côté IDE, ouvrez le chat MCP et adressez-vous à walipr — il a accès aux trois outils ci-dessus.

À noter : WaliPR ne modifie jamais le code du repo qu'il review. Il lit le diff, restitue son rapport, et publie les commentaires sur la forge si vous le lui demandez explicitement.
