SherleKhomes

Analyste de tickets Jira

« Quand un ticket arrive sur mon bureau, je l'épluche jusqu'à ce qu'il livre la vérité. »

Compatible : Claude Code Copilot Cursor

Mission

SherleKhomes prend un ticket Jira en entrée et restitue un rapport d'enquête complet : cause racine, méthode de reproduction, fix proposé (ou appliqué quand la confiance l'autorise), zones à risque de régression, et scores de confiance.

Capacités

  • analyze_ticket Diagnostic complet d'un ticket Jira (legacy / monolithique).
  • best_handoff Délègue à votre LLM avec un contexte enrichi prêt à l'emploi.
  • summarize_thread Résume description et commentaires en une fiche d'enquête.

Configurer votre agent

Renseignez votre Jira pour générer une installation personnalisée. Aucune donnée ne quitte ce navigateur : tout est calculé localement.

Type de Jira

L'URL d'accueil de votre Jira (sans /browse, sans clé de ticket).

Le token API correspondant sera saisi plus tard, en local.

ⓘ

Vos secrets (token, mot de passe) ne sont jamais demandés par cette page. Ils resteront dans ~/.bughound/.env, sur votre machine.

Une seule commande à coller dans votre terminal. Elle installera l'agent et créera ~/.bughound/config.yaml avec vos paramètres.

Installation manuelle ? 📥 Télécharger la wheel v1.0.2

Si vous préférez gérer la config à la main, déposez ce fichier dans ~/.bughound/config.yaml.

Bonne nouvelle : install.sh enregistre déjà BugHound comme serveur MCP dans Claude Code, VS Code (Copilot Chat) et Cursor — vous n'avez rien à coller. Pour ré-enregistrer manuellement (autre poste, ré-installation, override) : 

bughound mcp-install

Options utiles : --client claude|vscode-copilot|cursor pour cibler un seul client, --print-only pour voir le JSON sans rien écrire, --yes --overwrite pour un run idempotent (CI / scripts).

Si vous préférez éditer les fichiers à la main, voici le JSON attendu pour chaque client. Remplacez /usr/local/bin/bughound par le chemin absolu renvoyé par which bughound sur votre poste — VS Code lance les serveurs MCP avec un PATH minimal, le chemin absolu est obligatoire.

Fichier : ~/.claude.json (clé mcpServers). 

{
  "mcpServers": {
    "sherlekhomes": {
      "command": "/usr/local/bin/bughound",
      "args": ["mcp-serve"]
    }
  }
}

Fichier : ~/.config/Code/User/mcp.json (clé servers, format VS Code natif depuis 2025). Relancer VS Code après modification. 

{
  "servers": {
    "sherlekhomes": {
      "type": "stdio",
      "command": "/usr/local/bin/bughound",
      "args": ["mcp-serve"]
    }
  }
}

Fichier : ~/.cursor/mcp.json (clé mcpServers). 

{
  "mcpServers": {
    "sherlekhomes": {
      "command": "/usr/local/bin/bughound",
      "args": ["mcp-serve"]
    }
  }
}

Vos secrets — ~/.bughound/.env

install.sh crée ~/.bughound/.env avec les bonnes lignes commentées. Décommentez et collez vos vrais tokens — les commentaires # en tête de ligne sont obligatoires tant que la valeur n'est pas remplie. 

# Token Jira (si JIRA_MODE=api). Generation : Profil Atlassian
# → Security → API tokens. Pour Server/DC en mode browser, pas
# besoin : SherleKhomes utilise votre session Chrome.
# JIRA_API_TOKEN=

# Artifactory (téléchargement des artefacts liés au ticket).
# Token : interface Artifactory → User Profile → Generate API Key.
# ARTIFACTORY_USER=jdoe@maboite.com
# ARTIFACTORY_TOKEN=

# Clé du LLM (analyse). Anthropic, OpenAI, ou compatible.
# LLM_API_KEY=

Le fichier appartient uniquement à votre utilisateur (chmod 600 recommandé). N'envoyez jamais ce fichier dans un dépôt git ni dans un canal Slack.

Premiers pas

  1. Vérifiez vos secrets dans ~/.bughound/.env (bloc ci-dessus).
  2. Lancez votre première enquête : bughound analyze MON-TICKET-42
  3. Relancez votre IDE (VS Code / Cursor / Claude Code) et parlez à l'agent sherlekhomes depuis le chat MCP.

⚠️ Piège shell — variables côté pipe

Si vous tapez la commande à la main, les variables JIRA_URL / JIRA_MODE doivent être attachées au bash, pas au curl. Sinon install.sh ne les voit pas et ~/.bughound/.env n'est pas pré-rempli. 

# ❌ FAUX — variables attachées à curl, perdues après le pipe
JIRA_URL='https://jira.maboite.com' JIRA_MODE='browser' \
  curl -fsSL https://slama-consulting.github.io/agence/install.sh | bash

# ✅ CORRECT — variables attachées à bash, lues par install.sh
curl -fsSL https://slama-consulting.github.io/agence/install.sh \
  | JIRA_URL='https://jira.maboite.com' JIRA_MODE='browser' bash

# ✅ ÉQUIVALENT — export d'abord
export JIRA_URL='https://jira.maboite.com' JIRA_MODE='browser'
curl -fsSL https://slama-consulting.github.io/agence/install.sh | bash

Depuis la version 1.0.1 d'install.sh, si vous êtes sur un terminal interactif, le script lance bughound setup et vous demande directement les valeurs — le piège est neutralisé sur votre poste.

Installation rapide (sans configuration)
